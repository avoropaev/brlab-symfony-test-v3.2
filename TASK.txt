Введение
Приложение по REST API получает следующие данные о спортивных событиях (матчах):
* Язык (например: русский, ru, en, eng)
* Вид спорта (например: футбол, futbol, football)
* Лига (например: Лига чемпионов УЕФА, Liga UEFA, League UEFA)
* Команда 1 (например: Реал, Реал Мадрид, Real, Real M)
* Команда 2 (например: Барселона, ФК Барселона, Barcelona)
* Время начала матча (например, 2019-01-01 01:01:01)
* Источник (например: sportdata.com, anotherdata.com)


Задачи приложения
1. сохранять входящие данные в БД в таблицу с буферными матчами game_buffer
2. соотносить буферные матчи из разных источников c реальными матчами из таблицы game по следующим условиям:
   1. если в game не существует матча, с которым можно смерджить запись из  game_buffer - то создается новая запись в game
   2. если матч уже существует в game - смерджить матч из game_buffer с этим матчей.
3. отдавать с помощью REST API статистику о случайном матче из game, и количестве буферных матчей из game_buffer, смердженных с ним, с возможностью указать фильтры по источнику и интервалу начала матча.


Примечания
1. Матчи в game должны быть уникальными.
2. Уникальным является матч, в конкретной лиге между командой 1 и командой 2 в определенное время, например все дальше один и тот же матч:
   * Лига чемпионов УЕФА, Реал - Барселона, 2019-01-01 13:00:00
   * Лига УЕФА, Реал Мадрид - ФК Барселона, 2019-01-01 13:00:00
   * League UEFA, Real - Barcelona, 2019-01-01 13:00:00
   * League UEFA, Real M - Barcelona FC, 2019-01-01 13:00:00
3. Нужно учесть, что время начала матча у разных источников может отличаться. Для футбола это +- 26 часов.
Например следующие матчи из разных источников должны быть смерджены в один реальный матч с временем начала 2019-01-02 10:00:00:
   * Лига чемпионов УЕФА, Реал - Барселона, 2019-01-01 13:00:00 из источника sportdata2.com
   * Лига чемпионов УЕФА, Реал - Барселона, 2019-01-02 10:00:00 из источника sportdata3.com
   * Лига чемпионов УЕФА, Реал - Барселона, 2019-01-02 10:00:00 из источника sportdata4.com
4. Лига может быть связана только с 1 видом спортом
5. Команда может быть связана только с 1 видом спортом. Т.е. ЦСКА в Футболе и ЦСКА в Хоккее - это разные команды.
6. Количество записей в БД может быть очень большим
7. События из источников могут приходить не только по одному, но и пакетно
8. Одно и тоже событие может неоднократно приходить от одного и того же поставщика


Условия
1. Symfony 4
2. установка фреймворка должна быть отдельным коммитом
3. можно выбрать любую базу данных
4. приложение должно разворачиваться через `docker-compose up -d`
5. решение нужно залить на гитхаб
